Exemplos de empresas que usam o paradigma funcional

Diferentemente das programações imperativa e orientada a objetos, a programação funcional parte do princípio de que tudo são funções. Não existe uma lista de instruções ou objetos para o computador executar, mas uma sequência de funções matemáticas que, juntas, resolvem um problema.

"Um exemplo é você pintar um carro usando a programação orientada a objetos; ele vai terminar com uma cor diferente e sem registro de ter tido outra cor antes". No paradigma funcional, é como se você terminasse com um carro com a cor nova, mas ainda mantivesse outro com a cor antiga.

Nubank

O Nubank foi fundado em 2013, e a empresa procurou tecnologias que ajudassem a atingir suas metas empresariais, mantendo a eficiência e permitindo um crescimento seguro e sustentável. Naquele momento, o paradigma funcional se mostrou a melhor escolha para os desafios que enfrentavam.

O princípio de imutabilidade do paradigma funcional é um aliado crucial para uma instituição financeira, pois garante um registro de todas as informações. Além disso, é possível voltar no tempo e recuperá-las caso algo seja computado de forma incorreta.

O paradigma funcional também contribui para o processo de internacionalização do Nubank. Como é um paradigma imutável, não é necessário mudar tudo desde o início. A lógica do sistema financeiro de um país pode ser diferente, mas a empresa consegue usar a mesma base para todos.

Dessa forma, eles conseguem aproveitar todas as vantagens da programação funcional para construir soluções que devolvem às pessoas o controle sobre o seu dinheiro – independentemente do país.

Código em Python que se utiliza o paradigma funcional

        
        varMutable = 3
        print(varMutable) # 3


        # Função pura: não altera estado externo, apenas retorna um novo valor
        def pure_function(value):
            return value + 1


        #Cria uma nova variável com o novo valor
        new_var = pure_function(varMutable)


        print(varMutable) #3 (permanece imutável)
        print(new_var) #4 (novo valor retornado pela função pura)

Explicando o Código

Imutabilidade: A variável varMutable nunca é alterada. A função pure_function recebe varMutable como argumento e retorna um novo valor (sem modificar o original).

Função pura: A função pure_function é pura, pois não tem efeitos colaterais e não modifica o estado global, apenas retorna um novo valor.

Com isso, você segue o paradigma funcional e mantém o estado imutável no código.
